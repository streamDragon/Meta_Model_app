# Meta Model Feature Design

Use this skill when the task is not just "build a screen", but "build a thinking process into the screen".

## Goal
Design or improve features that help the user move from a surface sentence into a clearer internal structure:
- linguistic structure
- missing information
- category detection
- logical relations
- possible next explorations
- therapeutic or training usefulness

## Core design principle
Every Meta Model feature must answer 4 things clearly:
1. What is the original sentence or stimulus?
2. What exactly is being detected or explored?
3. What is the user supposed to do step by step?
4. What useful insight or action comes out at the end?

## Feature design rules
- The original sentence must remain visible or easily recoverable.
- The user must always know what they are doing now.
- Avoid dumping too many categories at once unless comparison is the point.
- Prefer progressive disclosure over full complexity on the first screen.
- If the feature has multiple layers, make the transition between layers explicit.
- Show why this exploration matters, not only what button to press.
- Every interaction should produce a meaningful shift: classify, reveal, compare, expand, separate, rank, or choose.
